# Sugcar Final-Slide Contact CTA Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a compact, accessible contact invitation to the final slide that opens `sugcar.app@gmail.com` through a new-tab `mailto:` action.

**Architecture:** Extend the existing closing-slide renderer in `src/main.js` with semantic CTA markup and style it through the existing closing-card rules in `styles.css`. Keep all content static and client-only; no form, backend, event tracking, or new dependency is introduced.

**Tech Stack:** HTML generated by JavaScript, CSS, Node.js built-in test runner.

## Global Constraints

- The CTA appears only on the final slide.
- The destination is exactly `sugcar.app@gmail.com`.
- The link uses `mailto:sugcar.app@gmail.com`, `target="_blank"`, and `rel="noopener noreferrer"`.
- The visible copy is “Questions or interested in testing Sugcar?”, “Email me”, and “Please don’t include personal health information.”
- The CTA is visually secondary to the closing statement and stacks on narrow screens.
- No form, backend, tracking event, independent animation, or persistent footer is added.
